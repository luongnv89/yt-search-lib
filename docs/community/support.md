## üêõ Reporting Bugs

### Before Reporting

1. **Check existing issues** - Your issue might be solved
2. **Read documentation** - Answer might be in docs
3. **Search discussions** - Someone might have asked already
4. **Try latest version** - Bug might be fixed

### When Reporting

Include:
- Minimal reproducible example
- Error message or unexpected behavior
- Environment details
- Expected vs actual result

### After Reporting

- Monitor your issue for questions
- Respond promptly to requests
- Be patient - volunteers maintain this
- Thank maintainers for help

## üí° Feature Requests

### Good Feature Request

1. **Clear description** - What should it do?
2. **Use case** - Why do you need it?
3. **Examples** - How would you use it?
4. **Alternatives** - Have you tried anything?

### Feature Request Process

1. Create [Feature Request Issue](https://github.com/luongnv89/youtube-search/issues/new?template=feature_request.md)
2. Community discusses
3. Maintainers evaluate
4. Implement if approved
